## 1.什么是数据？

- 存储在内存中，代表特定信息的东西，本质上是0101...
- 特点：可传递，可运算
- 一切皆数据

## 2.什么是内存？

- 内存条通电以后产生的可存储数据的空间（临时的）
- 内存产生和死亡：内存条（电路板）=>通电=>产生内存空间=>存储数据=>处理数据=>断电=>内存空间和数据都消失
- 一块小内存的2个数据
  - 内部存储的数据
  - 地址值
- 内存分类
  - 栈：全局变量/局部变量
  - 堆：对象

## 3.什么是变量？

- 可变化的量，由变量名和变量值组成
- 每个变量都对应一块小内存，变量名用来查找对应的内存

## 4.内存，数据，变量三者之间的关系

- 内存用来存储数据的空间
- 变量是内存的标识（用来查找对应的内存空间）

## 5.关于赋值和内存的问题

- 问题：var a = xxx，a内存中到底保存的是什么？

  - xxx是基本数据类型，a内存中保存的就是这个数据
  - xxx是对象，a内存中保存的就是对象的地址值
  - xxx是一个变量，保存的是xxx的内存内容，可能是基本数据类型，也可能是地址值

  ```js
  var a = 3
  a = function(){
  
  }
  
  var b = 'cjj'
  a = b //这里a是基本数据类型
  b = {} 
  a = b //这里a是地址值
  ```

## 6.关于引用变量赋值问题

- n个引用变量指向同一个对象，通过其中一个变量修改对象内部数据，另一个变量看到的是修改之后的数据
- 2个引用变量指向同一个对象，让其中一个引用变量指向另一个对象，另一个引用变量已然指向前一个对象

```js
var obj1 = {name: 'Tom'}
var obj2 = obj1
obj1.name = 'Cjj'
console.log(obj2.name) // 'Cjj'

var a = {age:12}
var b = a
a = {name: 'CJJ',age:13}
console.log(b.age,a.name,a.age)//14 CJJ 13

a = {name: 'CJJ',age:13}
function fn2(obj){
    obj = {age:15}
}
fn2(a)//函数执行完，函数内部的局部变量会自动释放，obj={age:15}就变成了垃圾对象
console.log(a.age)//13

//问题：在js调用函数时传递变量参数时，是值传递还是引用传递
//理解：传的是值，就是值传递；传的是地址值，就是引用传递
```

## 7.js引擎如何管理内存？

- 内存生命周期
  - 分配小内存空间，得到它的使用权
  - 分配数据，可以反复进行操作
  - 释放当前的小内存空间
- 释放内存
  - 局部变量：函数执行完自动释放
  - 对象：先成为垃圾对象，在后面由垃圾回收器回收
  - 全局变量不会释放

```js
var a = 3
var obj = {}
obj = null // 把obj变成垃圾对象

function fn(){
    var b = {}
}

fn() // b是自动释放，b所指向的对象是在后面的某个时刻由垃圾回收器回收
```
